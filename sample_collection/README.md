# 표본매장 운영설정 수집 (일회성)

브랜드별 표본매장의 **배민/쿠팡이츠 운영설정**(광고설정·할인/쿠폰·운영시간·최소주문금액·배달비·메뉴/가격)을
화면 그대로 저장한 뒤, 그 캡처를 Claude가 정밀 파싱하여 **matetech 일별 매출과 결합한 표본 데이터**를 만든다.

배민/쿠팡 로그인은 **사용자가 직접** 한다(2FA 포함). 이 스크립트는 로그인된 화면을 **저장만** 한다.

---

## 0. 준비 (사용자 PC, 한 번만)

```bash
cd sample_collection
pip install -r requirements.txt
playwright install chromium
```

> Python 3.9+ 권장. 회사 PC라 pip 설치가 막히면 개인 PC에서 진행해도 된다.

선택: `sample_stores.example.json` 을 `sample_stores.json` 으로 복사해 표본매장 목록을 적어두면
나중에 매출 결합 시 매장명 매칭이 쉬워진다. (캡처 단계에서는 필수 아님 — 스크립트가 매장명을 직접 물어본다.)

---

## 1. 캡처 실행

```bash
python capture.py
```

크롬 창이 뜬다. 매장마다 아래를 반복한다.

1. 터미널에 **브랜드명 / 매장명 / 플랫폼**(배민 or 쿠팡) 입력
2. 뜬 브라우저에서 **직접 로그인**(아이디·비번·SMS 2FA 등)
3. 캡처할 설정 페이지로 이동 (예: 배민 → 광고관리 화면)
4. 터미널에서 **Enter** → 라벨 선택(1~6 번호) → 현재 화면이 저장됨
   - 저장 항목 추천: `광고설정`, `할인쿠폰`, `운영시간`, `최소주문금액_배달비`, `메뉴가격`, `가게정보`
5. 그 매장의 항목을 다 캡처했으면 터미널에 **`next`** → 같은 매장의 다른 플랫폼이거나 다음 매장
6. 전부 끝나면 **`quit`**

> 한 매장에서 배민·쿠팡 둘 다 모으려면: 배민으로 매장 1회 캡처(next) → 다음에 같은 매장명/쿠팡 선택해서 한 번 더.

### 저장되는 것 (매장 폴더마다)
```
captures/<실행시각>/<브랜드>/<매장>__<플랫폼>/
  광고설정__HHMMSS.html        # 화면 HTML (JS 렌더링 포함)
  광고설정__HHMMSS.png         # 전체 스크린샷
  manifest.json                # 무엇을/어디서 캡처했는지 목록
  network/                     # 화면이 불러온 JSON 응답(정확한 수치가 들어있는 경우가 많음)
```

HTML·스크린샷·network JSON 세 가지를 같이 저장하므로, 어느 하나가 비어도 다른 걸로 파싱할 수 있다.

---

## 2. Claude에게 전달

`captures/<실행시각>` 폴더를 **통째로 zip** 으로 묶어서 이 세션(또는 새 세션)에 올린다.

```bash
# 예시
cd sample_collection/captures
zip -r 표본캡처.zip 20260604_xxxxxx
```

그러면 Claude가:
- 캡처된 HTML/JSON에서 **광고율·할인·쿠폰·운영시간·최소주문금액·배달비·메뉴가격**을 매장별로 정확히 추출
- matetech 일별 매출(기존 `sales_report.py` 로직 재사용)과 매장명 기준으로 결합
- **구글 시트**(또는 엑셀)로 `날짜 | 브랜드 | 매장 | 운영설정값 | 매출 | 건수 | 객단가` 표본 데이터 생성

---

## 자주 묻는 점

- **개인정보/보안**: 캡처 파일에는 화면에 보이는 내용(매출, 정산 등)이 그대로 담길 수 있다. 필요 없는 페이지는 캡처하지 말고, 전달 전에 민감 화면은 빼도 된다.
- **로그인이 자꾸 풀린다**: 매장별 개별계정이라 매장마다 새 세션으로 연다. 정상이다.
- **셀렉터를 안 쓰는 이유**: 화면을 통째로 저장하므로 사이트가 바뀌어도 캡처는 성공한다. 정확한 값 추출은 캡처를 받은 뒤 Claude가 한다.
