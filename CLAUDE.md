# 매출 리포트 자동 발송 — 일시 정지 안내

## 현재 상태

**매출 리포트 자동 발송은 일시 중단되어 있습니다.**

matetech가 정식 API 제공을 시작해서, 기존 Playwright(웹 스크래핑) 기반 코드를
API 호출 방식으로 전환하는 작업이 예정되어 있습니다. 그동안 자동/수동 실행 모두
멈춰둡니다.

## 트리거로 이 세션이 자동 시작된 경우

- **`sales_report.py`를 실행하지 마세요.** 실행되어도 `SystemExit(0)` 가드로 즉시 종료됩니다.
- **크롬(Playwright chromium)을 띄우지 마세요.** 로그인 시도 자체가 불필요합니다.
- **matetech 로그인 페이지 접속 / 계정 정보 사용을 하지 마세요.**
- 스케줄 트리거로 세션이 자동 시작됐다면, 아무 작업도 하지 말고 "일시 정지 상태입니다"라고
  안내한 뒤 종료하세요.

## 다시 켜는 방법 (사용자가 명시적으로 요청할 때만)

1. `sales_report.py`의 `if __name__ == "__main__":` 아래 "일시 정지 가드" 3줄 제거
2. `.github/workflows/sales_report.yml`의 `schedule:` 블록 주석 해제
3. code.claude.com 트리거는 사용자가 별도로 관리

## 강제 실행 (테스트 목적)

정말 필요하면 환경변수 `FORCE_RUN_SALES_REPORT=true` 를 주면 스크립트가 돌아갑니다.
자동 트리거에서는 절대 이 변수를 세팅하지 마세요.
