# 월마감 정산 페이지

브랜드별·매장별 **매출 / 전용물류매출 / 전용이익**을 전월대비로 정리하는 단일 HTML 페이지입니다.
서버 없이 브라우저에서 동작하며, 업로드한 데이터는 외부로 전송되지 않습니다.

## 사용법

1. `index.html`을 브라우저로 엽니다. (더블클릭 또는 GitHub Pages 호스팅)
2. 파일 3종을 업로드합니다.
   - **당월 매출 로우데이터** (필수): `매장코드 · 매장명 · 총매출액` 컬럼 포함
   - **당월 물류매출** (필수): `매장코드 · 매장명 · 전용상품발주금액 · 본사이익` 컬럼 포함
   - **전월 결과파일** (선택): 지난달에 "결과 엑셀 내보내기"로 받은 파일
3. **정산 계산** 버튼을 누르면 브랜드별 요약과 매장별 상세표가 나옵니다.
4. 필요하면 매장별 **사입** 체크박스로 사입매장을 지정합니다. (지정은 브라우저에 저장되어 다음에도 유지됩니다.)
5. **결과 엑셀 내보내기**로 저장하면, 다음 달에 그 파일을 "전월 결과파일"로 올려 전월대비를 볼 수 있습니다.

## 계산식

| 항목 | 정의 |
|---|---|
| 매출 | 매출 로우데이터의 총매출액 |
| 전용물류매출 | 물류매출의 전용상품발주금액 |
| 전용이익 | 물류매출의 본사이익 |
| 전용발주율 | 전용물류매출 ÷ 매출 |
| 전용이익률 | 전용이익 ÷ 전용물류매출 |
| 전월대비 (금액) | (당월 − 전월) ÷ 전월 (%) |
| 전월대비 (비율) | 당월 − 전월 (%p) |

- 브랜드는 매장명 접두어(`밥구정로데오_서울영등포점` → **밥구정로데오**)로 자동 분류합니다.
- 매출·물류 데이터는 **매장코드**로 연결합니다. 한쪽에만 있는 매장은 표에 `물류X` / `매출X` 배지로 표시합니다.
- **사입매장(-7.5%)**: 사입으로 지정한 매장은 배지로 표시하며, 합계에는 영향을 주지 않습니다(표시 전용). 참고용으로 `매출 × (1−7.5%)` 값을 함께 보여줍니다.

## 컬럼 자동 인식

헤더 이름의 키워드로 컬럼을 자동 인식합니다(예: `총매출액`, `전용상품발주금액`, `본사이익`).
컬럼명이 달라도 위 키워드가 포함되면 인식되며, `~율`/`~률`/`~대비` 같은 비율 컬럼은 금액으로 오인하지 않도록 제외합니다.
