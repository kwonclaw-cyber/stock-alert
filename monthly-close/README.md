# 월마감 정산 페이지

브랜드별·매장별 **매출 / 전용물류매출 / 전용이익**을 전월대비로 산출하는 단일 HTML 페이지입니다.
서버 없이 브라우저에서 동작하며, 업로드한 데이터는 외부로 전송되지 않습니다.
기존에 수기로 쓰던 "월마감 엑셀(브랜드관리팀 SV월보)"의 계산식을 그대로 구현했습니다.

## KPI 관리 (맨 위)

페이지 최상단 KPI 카드로 목표 대비 현황을 한눈에 봅니다(달성/근접/하락 색상).
- **육식사관학교**: 점당매출 전월대비 +5%↑ — 현재 점당매출·전월대비 자동 표시
- **밥구정로데오**: 점당매출 현상태 유지 — 현재 점당매출·전월대비 자동 표시
- **제주할멍고기국수**: 점당매출 1,200만원 — 현재 점당매출 직접 입력(저장), 목표 대비 표시

브랜드별 요약은 큰 숫자·증감 pill·당월/전월 막대그래프·지표 그리드로 도식화했습니다.

## 특이사항 · 강조 메모

KPI 아래 "특이사항 · 강조"에 내용을 입력하고 구분(📌강조/⚠️주의/ℹ️정보/📝기본)·제목을 골라 **카드 추가**하면,
색상으로 꾸며진 카드로 표시됩니다. 카드는 브라우저에 저장되며 ✕로 삭제, 결과 엑셀의 `특이사항` 시트로도 저장됩니다.

## 바로 쓰는 법 (가장 간단)

`월마감.html` **한 파일만 받아 더블클릭**하면 됩니다. 인터넷 연결·설치 불필요,
모든 처리가 브라우저 안에서만 이루어져 매출·매입가 등 데이터가 외부로 전송되지 않습니다.
(SheetJS 라이브러리와 매입가·매장 매칭표가 파일 안에 모두 내장돼 있습니다.)

> 개발/유지보수용 분리본은 `index.html` + `vendor/` + `data/` 폴더입니다.
> 내장 데이터(`data/*.js`)나 화면(`index.html`)을 수정한 뒤
> `python3 build_single.py` 를 실행하면 `월마감.html` 단일 파일이 다시 생성됩니다.

## 매월 업로드하는 파일

| 구분 | 내용 | 비고 |
|---|---|---|
| **POS 매출 (브랜드별)** | matetech "기간별 매출분석 매장별분리" 내보내기 | 밥구정·육식 등 여러 개 한 번에 선택 |
| **물류 발주원장 (누적)** | 푸드가이드 "상품별 실적조회" 누적 원장 | 발주일·브랜드·전/범용·공급가·총합계 포함 |
| 전용상품 매입가 (선택) | 신규 전용상품 매입가 보충용 | `매입가?` 경고가 뜰 때만 |
| 전월 데이터 (선택) | 지난달 "결과 내보내기" 파일 **또는 전월 월마감(브랜드관리) 파일** | 전월대비·당월오픈 판정용 |

업로드 후 **대상월(당월)** 을 고르면(기본=물류원장 최신월) 자동 계산합니다.
전월(전월대비)은 물류원장의 직전월 데이터로 자동 비교하며, 매출 전월대비는 전월 결과파일이 있어야 표시됩니다.

## 계산식 (월마감 엑셀과 동일)

| 항목 | 정의 |
|---|---|
| 매출 | POS의 선택 컬럼 (기본 **순매출액**, VAT제외). 실매출액·매출액으로 변경 가능 |
| **전용물류매출** | 물류원장에서 `전/범용 = 전용` 행의 **총합계(공급가+부가세)** 합 (매장별) |
| **전용이익(본사이익)** | 전용 행의 **(공급가 − 매입가×수량 − 공급가×물류수수료율)** 합 |
| 물류수수료 | 공급가 × **8.2%** (페이지에서 변경 가능) |
| 매입가 | 내장 매입가 마스터(`data/buyprice.js`)에서 상품코드로 조회 |
| 전용발주율 | 전용물류매출 ÷ 매출 |
| 전용이익률 | 전용이익 ÷ 전용물류매출 |
| 전월대비(금액) | (당월 − 전월) ÷ 전월 (%) |
| 전월대비(비율) | 당월 − 전월 (%p) |

검증: 원본 월보(202604)와 매장별 본사이익이 정확히 일치합니다
(예: 광주서구본점 1,528,332 / 영등포 1,284,498).

## 사입매장 자동 분류

각 브랜드 **기준매장**의 전용발주율보다 임계값(기본 **7.5%p**) 이상 낮은 매장을 사입으로 분류합니다.

- 기본 기준매장: 밥구정로데오 = `광주서구본점`, 육식사관학교 = `남양읍점`
- 기준매장·임계값은 페이지에서 변경 가능(브라우저에 저장)

## 매장 매칭 관리 (POS ↔ 물류 코드 연결)

POS(메이트코드·POS 매장명)와 물류(물류코드·동원 매장명)는 코드 체계와 명칭이 달라
내장 매칭표(`data/storematch.js`)로 연결합니다. 신규 매장 등 매칭표에 없는 매장은
`매출X` 로 표시되며, 화면 상단 **"매장 매칭 관리"** 패널에 나옵니다.

- 매출이 연결되지 않은(또는 매출 0원) 매장은 **브랜드 리스트·합계에서 제외**되고, 이 패널에만 표시됩니다.
- 미매칭 물류 매장마다 연결할 POS 매장을 고르는 드롭다운 제공 (매장명 유사도로 **추천 자동 선택**) → 매칭 저장 시 리스트에 포함
- **매칭 저장**을 누르면 `물류코드 → 메이트코드` 매핑이 **브라우저에 영구 저장**되어 다음 달부터 자동 적용
- 모든 사용자/기기에 영구 반영하려면 `data/storematch.js`(내장 매칭표)를 갱신해 커밋

## 이번 달 마감 저장 → 다음 달 전월자료 자동 사용

5월 마감을 끝낸 뒤 **"이번 달 마감 저장"** 버튼을 누르면 그 결과가 브라우저에 저장됩니다.
다음 달(6월) 마감 때 전월 파일을 따로 올리지 않아도, **저장된 5월 마감이 전월자료로 자동 적용**됩니다.
(전월 파일을 직접 올리면 그게 우선합니다. 저장된 월 목록은 버튼 옆에 표시됩니다.)

## 전월 데이터 · 당월 오픈 구분

전월 데이터는 두 가지를 모두 인식합니다: 도구의 "결과 내보내기" 파일, 또는 **전월 월마감(브랜드관리) 파일**.
브랜드관리 파일을 올리면 그 파일의 `당월` 컬럼(=직전월)에서 매장별 **매출·전용물류매출·전용이익**을 가져와 전월대비합니다.

- **당월 오픈**: 전월 데이터에 없는 매장(양도양수 통합 후 기준)은 `당월오픈` 으로 구분 표시(초록 배지·구분=오픈)하고 브랜드별 오픈 수를 집계합니다.
- 양도양수로 코드가 바뀐 매장은 통합되면 전월(구코드)과 연결되어 **오픈으로 잡히지 않습니다.**

## 양도양수 매장 통합 (코드가 갈린 동일 매장 합산)

양도양수로 사업자/코드가 바뀐 동일 매장(예: `봉천점` → `봉천점(2)`, `서울독산점` → `독산점(2)`)을
하나로 묶어 **당월·전월 모두 합산**하고 전월대비를 연속되게 처리합니다.

- **높은 신뢰 그룹은 저장 없이 바로 자동 적용**됩니다 (지역명이 같거나 `(2)`·`(3)` 재등록 표기가 있는 경우 → `자동` 표시).
- 낮은 신뢰(지역명만 같고 표기 단서 없음)는 `확인` 표시로 **기본 미적용** — 같은 매장이면 체크 후 저장.
- 잘못 묶일 만한 후보(예: `광주동구점`↔`대전동구점`은 다른 매장)는 기본 미적용이라 그대로 두면 됩니다.
- 체크/해제·대표(최신 코드) 변경 후 **통합 저장**을 누르면 그 결정이 브라우저에 저장되어 자동 기본값을 덮어씁니다.
- `결정 초기화`를 누르면 사용자 결정이 지워지고 높은 신뢰 자동 적용 상태로 돌아갑니다.

## 내장 참조 데이터 (`data/`)

- `buyprice.js` — 전용상품 매입가 마스터 (상품코드 → 매입가)
- `storematch.js` — 매장 매칭표 (메이트코드 ↔ 물류코드 ↔ 매장명 ↔ 브랜드)

> **신규 전용상품 매입가 (추후 추가 반영)**: 어떤 상품이 새로 "전용"으로 전환되면 매입가 마스터에 없을 수 있습니다.
> 이때 매장에 `매입가?` 배지와 상단 경고가 표시되며, 해당 전용이익이 과대 계산됩니다. 보충 방법:
> 1. **화면에서 직접 입력** — 경고에 누락 상품(코드·상품명·관련 공급가)이 표로 나오며, 각 행의 칸에 매입가를 입력하고 저장하면 즉시 반영됩니다.
> 2. **파일 업로드** — `상품코드·매입가` 컬럼이 든 파일을 "전용상품 매입가" 칸에 올리면 한 번에 보충됩니다.
>
> 추가한 매입가는 **브라우저에 영구 저장**되어 다음에 열 때도 자동 적용됩니다(같은 브라우저 기준).
> 모든 사용자/기기에 영구 반영하려면 `data/buyprice.js`(내장 마스터)를 갱신해 커밋하세요.

## 추가 매출 비교 (직접 입력)

물류/POS 계산과 무관하게 **매출만 비교**하는 항목입니다.

- **샵인샵 매출**: 물류 발주원장에서 브랜드가 `샵인샵`·`신규샵인샵`인 품목의 **총합계(공급가+부가세) 합 × 150%** 로 자동 계산(당월/전월, 읽기전용). 대상월은 상단 대상월 선택을 따릅니다.
- **제주할멍 매출** 등 그 외 항목: 당월·전월 매출을 직접 입력하면 전월대비(%)가 계산됩니다. `+ 항목 추가`로 줄 추가 가능.
- 입력값은 브라우저에 저장되며, 결과 엑셀의 `추가매출` 시트로 함께 저장됩니다.

## 수기 수정 (값 덮어쓰기)

"리스트 정렬·필터"의 **수기 수정 모드**를 켜면 매장별 **매출·전용물류매출·전용이익** 칸이 입력칸으로 바뀝니다.
값을 입력하면 그 매장의 수치를 덮어쓰고 **합계·비율·사입판정·전월대비에 반영**됩니다. 칸을 비우면 자동계산으로 복원됩니다.
수정값은 브라우저에 저장되며(대표 물류코드 기준), 수정된 셀은 `✎` 표시됩니다. `수기수정 초기화`로 전체 해제할 수 있습니다.

## 리스트 정렬·필터

매장 상세 리스트 위 "리스트 정렬·필터"에서:
- **정렬**: 매출 / 전용물류매출 / 전용이익 / 전용발주율 / 전용이익률 / 매장명 + 오름·내림차순
- **구분 필터**: 전체 / 사입 / 당월오픈 / 기준매장 / 일반

정렬·필터는 모든 브랜드 표에 함께 적용되며, 브랜드 합계는 전체 기준으로 유지됩니다(필터 시 "표시 N" 표기).

## PPT 발표자료 생성

정산 계산 후 **"📊 PPT 발표자료"** 버튼을 누르면 편집 가능한 PowerPoint(.pptx)가 생성됩니다.
표지 → 전체 요약(브랜드 비교표) → 브랜드별 주요지표 → KPI 현황 → 특이사항 순으로 구성됩니다.
(글꼴은 맑은 고딕 기준. PptxGenJS 내장)

## 결과 내보내기

`결과 엑셀 내보내기`로 저장하면 브랜드·매장별 전 항목이 담긴 파일이 생성됩니다.
이 파일을 다음 달 "전월 결과파일"로 올리면 매출 전월대비가 표시됩니다.
